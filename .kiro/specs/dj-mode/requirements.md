# Requirements Document

## Introduction

DJ Mode is an optional feature for the music game admin page that adds an AI-generated DJ voice-over between tracks. When enabled, the admin can configure how often the DJ speaks (Never / Rarely / Sometimes / Often / Always). Between each song, a short DJ script is generated via the Venice AI API and converted to audio using Venice AI's own TTS endpoint (`/api/dj-tts`), which calls `https://api.venice.ai/api/v1/audio/speech` using the same `VENICE_AI_API_KEY` already used for script generation. The script and audio are prefetched while the current track is still playing so the voice plays immediately when the song ends with zero delay. Playback of the next track is delayed until the DJ audio finishes, unless Duck & Overlay Mode is enabled — in which case the next song starts immediately at reduced volume while the DJ voice plays simultaneously, then ramps back to full volume when the voice finishes.

## Glossary

- **DJ_Mode**: The optional feature that triggers AI-generated DJ announcements between tracks.
- **Admin_Page**: The admin control panel at `app/[username]/admin/` where the operator manages the jukebox.
- **Venice_AI**: The third-party LLM API (venice.ai) used to generate DJ scripts.
- **Venice_TTS**: Venice AI's text-to-speech endpoint (`https://api.venice.ai/api/v1/audio/speech`) that accepts a text string and returns an MP3 audio file. Uses model `tts-kokoro` with voice `af_nova`. Accessed via the `/api/dj-tts` server-side route using the same `VENICE_AI_API_KEY`.
- **TTS_Audio**: The audio file returned by Venice_TTS for a given DJ_Script.
- **DJ_Script**: A short spoken announcement generated by Venice_AI that introduces the next track and artist.
- **AutoPlay_Service**: The singleton service (`AutoPlayService`) that manages track transitions and queue management.
- **DJService**: The singleton service (`DJService`) that manages DJ announcement logic, prefetching, and audio playback.
- **Next_Track**: The upcoming `JukeboxQueueItem` that will be played after the current track finishes.
- **VENICE_AI_API_KEY**: The environment variable holding the API key for authenticating with both Venice_AI (script generation) and Venice_TTS (audio synthesis).
- **DJ_Frequency**: The admin-configured setting controlling how often DJ announcements are triggered. Stored in `localStorage` as `"djFrequency"`. Valid values: `"never"` (0%), `"rarely"` (~10%), `"sometimes"` (~25%), `"often"` (~50%), `"always"` (100%).
- **Duck_Overlay_Mode**: An optional mode where the next track starts immediately at half volume while the DJ voice plays, then ramps back to full volume when the DJ voice finishes.
- **Prefetch**: The act of generating the DJ_Script and fetching the TTS_Audio during the current track's playback so both are ready before the track ends.

## Requirements

### Requirement 1: DJ Mode Toggle on Admin Page

**User Story:** As an admin, I want a DJ Mode checkbox on the admin page, so that I can enable or disable AI-generated DJ announcements between tracks.

#### Acceptance Criteria

1. THE Admin_Page SHALL display a checkbox labeled "DJ Mode" in the dashboard tab settings area.
2. WHEN the admin checks the DJ Mode checkbox, THE Admin_Page SHALL persist the DJ Mode enabled state to `localStorage` so it survives page refreshes.
3. WHEN the admin unchecks the DJ Mode checkbox, THE Admin_Page SHALL persist the DJ Mode disabled state to `localStorage` so it survives page refreshes.
4. THE Admin_Page SHALL display the current DJ Mode state (checked or unchecked) on load, reading the persisted value from `localStorage`.

---

### Requirement 2: Configurable DJ Announcement Frequency

**User Story:** As an admin, I want to configure how often the DJ speaks using preset options, so that I can tune the experience without hardcoded probabilities.

#### Acceptance Criteria

1. THE Admin_Page SHALL display a dropdown labeled "DJ Frequency" in the dashboard tab settings area, visible when DJ Mode is enabled.
2. THE DJ Frequency dropdown SHALL offer exactly five options: "Never" (0% probability), "Rarely" (~10% probability), "Sometimes" (~25% probability), "Often" (~50% probability), and "Always" (100% probability).
3. WHEN the admin selects a DJ Frequency option, THE Admin_Page SHALL persist the selection to `localStorage` under the key `"djFrequency"` so it survives page refreshes.
4. THE Admin_Page SHALL read the persisted `"djFrequency"` value from `localStorage` on load and display the matching option as selected.
5. IF no `"djFrequency"` value is present in `localStorage`, THEN THE Admin_Page SHALL default to "Sometimes" (~25% probability).
6. WHEN a track finishes and DJ Mode is enabled, THE DJService SHALL evaluate the probability corresponding to the current DJ_Frequency setting to determine whether to trigger a DJ announcement.
7. WHILE DJ Mode is disabled, THE DJService SHALL skip the probability check and play the Next_Track immediately without any announcement.
8. IF the Next_Track is not available in the queue when the probability check triggers, THEN THE DJService SHALL skip the DJ announcement and handle the empty queue normally.

---

### Requirement 3: DJ Script Generation via Venice AI

**User Story:** As an admin, I want the DJ script to be generated by an AI model, so that the announcements sound natural and informative.

#### Acceptance Criteria

1. WHEN a DJ announcement is triggered, THE DJService SHALL send a request to the Venice_AI chat completions API using the VENICE_AI_API_KEY from the environment.
2. THE DJService SHALL include the Next_Track's name and artist name in the prompt sent to Venice_AI.
3. THE DJService SHALL request a DJ script of no more than 3 sentences from Venice_AI.
4. IF the Venice_AI API request fails or returns an error, THEN THE DJService SHALL log the error and proceed to play the Next_Track without an announcement.
5. IF the VENICE_AI_API_KEY environment variable is not set, THEN THE DJService SHALL skip the DJ announcement and play the Next_Track immediately.
6. THE DJService SHALL use the `llama-3.3-70b` model when calling the Venice_AI API, prioritising low latency and cost.

---

### Requirement 4: Audio TTS Playback via Venice AI TTS

**User Story:** As an admin, I want the DJ script to be converted to audio by Venice AI's TTS endpoint and played back before the next track, so that guests hear a high-quality voice announcement.

#### Acceptance Criteria

1. WHEN a DJ_Script is successfully generated, THE DJService SHALL send the DJ_Script to the `/api/dj-tts` route and receive a TTS_Audio file in response.
2. THE DJService SHALL play the TTS_Audio file using an HTML `<audio>` element or the Web Audio API.
3. WHEN the TTS_Audio begins playing, THE DJService SHALL hold the track transition until the TTS_Audio finishes, unless Duck_Overlay_Mode is enabled.
4. WHEN the TTS_Audio finishes playing and Duck_Overlay_Mode is disabled, THE DJService SHALL immediately begin playing the Next_Track at full volume.
5. IF the `/api/dj-tts` request fails or returns an error, THEN THE DJService SHALL log the error and proceed to play the Next_Track without an announcement.
6. IF the `VENICE_AI_API_KEY` environment variable is not set, THEN THE `/api/dj-tts` route SHALL return a 500 error and THE DJService SHALL skip the TTS step and play the Next_Track immediately.

---

### Requirement 5: Environment Configuration

**User Story:** As a developer, I want the Venice AI API key to be defined in the environment file, so that credentials are not hardcoded in source code.

#### Acceptance Criteria

1. THE Admin_Page application SHALL read the Venice_AI API key exclusively from the `VENICE_AI_API_KEY` server-side environment variable.
2. THE application SHALL document the `VENICE_AI_API_KEY` variable in the `.env.example` file.
3. IF the `VENICE_AI_API_KEY` environment variable is absent or empty at runtime, THEN THE DJService SHALL treat DJ Mode as non-functional and skip all DJ announcement logic.

---

### Requirement 6: Prefetching DJ Script and Audio

**User Story:** As an admin, I want the DJ script and audio to be prepared while the current track is still playing, so that the voice announcement plays immediately when the song ends with zero delay.

#### Acceptance Criteria

1. WHEN a track begins playing and DJ Mode is enabled, THE DJService SHALL evaluate the DJ_Frequency probability and, if triggered, begin prefetching the DJ_Script and TTS_Audio for the Next_Track in the background.
2. WHILE the current track is playing, THE DJService SHALL fetch the DJ_Script from Venice_AI and the TTS_Audio from Venice_TTS concurrently so both are ready before the track ends.
3. WHEN the current track finishes and a prefetched TTS_Audio is available, THE DJService SHALL begin playing the TTS_Audio immediately without any additional network requests.
4. IF the prefetch has not completed by the time the current track ends, THEN THE DJService SHALL wait for the in-flight prefetch to complete before playing the TTS_Audio, rather than issuing a duplicate request.
5. IF the prefetch fails for any reason, THEN THE DJService SHALL log the error and proceed to play the Next_Track without an announcement.
6. THE DJService SHALL discard any prefetched data that is no longer relevant (e.g. the queue changed and the Next_Track is different) and skip the announcement for that transition.

---

### Requirement 7: Configurable DJ Frequency Persisted to Admin Dashboard

**User Story:** As an admin, I want the DJ frequency setting to be visible and editable on the admin dashboard, so that I can adjust it at any time without losing my preference.

#### Acceptance Criteria

1. THE Admin_Page SHALL display the current DJ_Frequency selection on the dashboard tab whenever DJ Mode is enabled.
2. WHEN the admin changes the DJ_Frequency dropdown, THE Admin_Page SHALL immediately persist the new value to `localStorage` under `"djFrequency"`.
3. THE Admin_Page SHALL reflect the persisted DJ_Frequency value on every page load without requiring the admin to reconfigure it.
4. WHEN DJ Mode is disabled, THE Admin_Page SHALL hide the DJ_Frequency dropdown to avoid confusion.

---

### Requirement 8: Duck & Overlay Mode

**User Story:** As an admin, I want a Duck & Overlay Mode toggle on the admin dashboard, so that the next song can start playing at reduced volume while the DJ voice plays simultaneously, creating a smooth radio-style transition.

#### Acceptance Criteria

1. THE Admin_Page SHALL display a toggle labeled "Duck & Overlay" in the dashboard tab settings area, independent of the DJ Mode toggle.
2. WHEN the admin enables Duck & Overlay Mode, THE Admin_Page SHALL persist the enabled state to `localStorage` under the key `"duckOverlayMode"` so it survives page refreshes.
3. WHEN the admin disables Duck & Overlay Mode, THE Admin_Page SHALL persist the disabled state to `localStorage` under the key `"duckOverlayMode"`.
4. THE Admin_Page SHALL read the persisted `"duckOverlayMode"` value from `localStorage` on load and display the toggle in the correct state.
5. WHEN a DJ announcement is triggered and Duck_Overlay_Mode is enabled, THE DJService SHALL start the Next_Track immediately at 50% volume while the TTS_Audio plays simultaneously.
6. WHEN the TTS_Audio finishes playing and Duck_Overlay_Mode is enabled, THE DJService SHALL ramp the Next_Track's volume from 50% back to 100% over a transition period of no more than 2 seconds.
7. WHILE Duck_Overlay_Mode is disabled, THE DJService SHALL use the default behavior of waiting for the TTS_Audio to finish before starting the Next_Track.
8. IF no DJ announcement is triggered for a given track transition, THE DJService SHALL play the Next_Track at full volume regardless of the Duck_Overlay_Mode setting.
9. THE Duck_Overlay_Mode setting SHALL operate independently of DJ Mode — the admin can enable or disable it without affecting the DJ Mode toggle state.
