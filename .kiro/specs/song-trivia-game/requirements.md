# Requirements Document

## Introduction

Replace the existing DGS (Dual Gravity System) music game with a song trivia game. When patrons visit the game page for a venue, they are presented with a multiple-choice trivia question about the currently playing song or artist. Questions and answer options are generated by Venice AI. Players score points for correct answers. When the song changes, a new question is generated. At the end of each clock hour, the top scorer is announced by the DJ (if DJ mode is enabled) and all scores reset. The game page is publicly accessible across multiple devices with no login required.

## Glossary

- **Game_Page**: The public-facing page at `/{username}/game` that displays the trivia game to patrons
- **Trivia_Engine**: The server-side API route that generates trivia questions and answer options using Venice AI
- **Score_Tracker**: The Supabase-backed system that records and aggregates player scores per venue
- **Player**: An anonymous patron participating in the trivia game on any device, identified by a browser-generated session ID
- **Venue**: A bar or restaurant identified by a unique username, with an associated `profile_id`
- **Now_Playing**: The Supabase table and Realtime subscription that tracks the currently playing song per venue
- **DJ_Announcer**: The existing DJ mode system that generates and plays voice announcements via Venice AI TTS
- **Venice_AI**: The external LLM API used for generating trivia questions (text) and DJ announcements (TTS)
- **Hourly_Reset**: The process that determines the winner, triggers a DJ announcement, and resets all scores at the top of each clock hour

## Requirements

### Requirement 1: Trivia Question Generation

**User Story:** As a patron, I want to see a trivia question about the currently playing song or artist, so that I can test my music knowledge while enjoying the venue.

#### Acceptance Criteria

1. WHEN a song is currently playing for a Venue, THE Trivia_Engine SHALL generate a trivia question related to the song title, artist, album, or genre using Venice_AI
2. THE Trivia_Engine SHALL return exactly 4 answer options, one of which is correct
3. THE Trivia_Engine SHALL randomize the position of the correct answer among the 4 options
4. WHEN Venice_AI fails to respond or returns an invalid response, THE Trivia_Engine SHALL return an error status with a descriptive message
5. THE Trivia_Engine SHALL include a `correctIndex` field (0–3) identifying the correct answer in the returned options array
6. WHEN the Trivia_Engine generates a question, THE Trivia_Engine SHALL return the question text, the 4 options, and the correct index in a single JSON response

### Requirement 2: Song Change Detection and Question Refresh

**User Story:** As a patron, I want a new trivia question every time the song changes, so that the game stays fresh and relevant to what is playing.

#### Acceptance Criteria

1. WHEN the `spotify_track_id` in the Now_Playing table changes for a Venue, THE Game_Page SHALL detect the change via Supabase Realtime subscription
2. WHEN a song change is detected, THE Game_Page SHALL request a new trivia question from the Trivia_Engine for the new song
3. WHILE a new question is being fetched, THE Game_Page SHALL display a loading indicator to the Player
4. WHEN no song is currently playing for a Venue, THE Game_Page SHALL display a waiting state informing the Player that the game will begin when music starts
5. WHEN a song change occurs, THE Game_Page SHALL clear the previous question and answer selection before displaying the new question

### Requirement 3: Answer Selection and Scoring

**User Story:** As a patron, I want to earn points for correct answers, so that I can compete with other patrons at the venue.

#### Acceptance Criteria

1. WHEN a Player selects an answer, THE Game_Page SHALL visually indicate whether the selected answer is correct or incorrect
2. WHEN a Player selects the correct answer, THE Score_Tracker SHALL increment the Player score by 1 point for that Venue
3. WHEN a Player selects an answer, THE Game_Page SHALL disable further answer selections for the current question
4. THE Game_Page SHALL display the Player current score persistently on screen
5. WHEN a Player selects an incorrect answer, THE Game_Page SHALL visually highlight the correct answer

### Requirement 4: Player Identity

**User Story:** As a patron, I want to enter a display name so that my score can be identified on the leaderboard and in winner announcements.

#### Acceptance Criteria

1. WHEN a Player first visits the Game_Page, THE Game_Page SHALL prompt the Player to enter a display name before joining the game
2. THE Game_Page SHALL store the Player display name and a generated session ID in browser local storage for session persistence
3. THE Game_Page SHALL validate that the display name is between 1 and 20 characters
4. WHEN a Player returns to the Game_Page with an existing session, THE Game_Page SHALL restore the Player display name and session ID from local storage
5. THE Game_Page SHALL allow the Player to change the display name at any time

### Requirement 5: Leaderboard Display

**User Story:** As a patron, I want to see a live leaderboard, so that I can see how I rank against other players at the venue.

#### Acceptance Criteria

1. THE Game_Page SHALL display a leaderboard showing all Players with scores greater than 0 for the current Venue
2. THE Game_Page SHALL order the leaderboard by score in descending order
3. WHEN a Player score changes in the Score_Tracker, THE Game_Page SHALL update the leaderboard in real time via Supabase Realtime
4. THE Game_Page SHALL visually highlight the current Player entry on the leaderboard
5. WHEN multiple Players have the same score, THE Game_Page SHALL order tied Players by earliest first-score timestamp

### Requirement 6: Hourly Score Reset and Winner Announcement

**User Story:** As a venue owner, I want scores to reset every hour with the winner announced by the DJ, so that the game stays competitive and engaging throughout the night.

#### Acceptance Criteria

1. WHEN the system clock reaches the top of a clock hour (XX:00:00), THE Score_Tracker SHALL determine the Player with the highest score for each active Venue
2. WHEN a winner is determined and DJ mode is enabled for the Venue, THE DJ_Announcer SHALL generate and play a voice announcement naming the winning Player and the winning score
3. WHEN the winner announcement is complete or DJ mode is disabled, THE Score_Tracker SHALL reset all Player scores for the Venue to 0
4. WHEN no Players have scored during the hour, THE Score_Tracker SHALL skip the winner announcement and reset silently
5. THE Game_Page SHALL display a countdown timer showing time remaining until the next hourly reset
6. WHEN scores are reset, THE Game_Page SHALL update the leaderboard in real time to reflect the cleared scores

### Requirement 7: Multi-Device Support

**User Story:** As a patron, I want to play the trivia game on my own phone, so that multiple people at the venue can play simultaneously.

#### Acceptance Criteria

1. THE Game_Page SHALL operate independently on each device without requiring authentication or login
2. WHEN multiple devices access the Game_Page for the same Venue, each device SHALL receive the same trivia question for the currently playing song
3. THE Game_Page SHALL use the `username` URL parameter to resolve the Venue `profile_id` via the existing `useProfileId` hook
4. WHEN a Player submits a score from any device, THE Score_Tracker SHALL persist the score to Supabase so all devices reflect the updated leaderboard

### Requirement 8: Trivia Question Caching

**User Story:** As a venue owner, I want trivia questions to be cached per song, so that Venice AI API calls are minimized and all players see the same question.

#### Acceptance Criteria

1. WHEN the Trivia_Engine generates a question for a song, THE Trivia_Engine SHALL store the question, options, and correct index in a Supabase `trivia_questions` table keyed by `profile_id` and `spotify_track_id`
2. WHEN a trivia question already exists in the cache for the current song and Venue, THE Trivia_Engine SHALL return the cached question instead of calling Venice_AI
3. THE Game_Page SHALL fetch the current trivia question from the cache table rather than calling the generation API directly, so all devices for a Venue see the same question
4. WHEN a song change is detected, THE Game_Page SHALL query the cache for the new song and request generation only if no cached question exists

### Requirement 9: Game Page Layout

**User Story:** As a patron, I want a clean and intuitive game interface, so that I can focus on answering trivia questions.

#### Acceptance Criteria

1. THE Game_Page SHALL display the current song name and artist name at the top of the screen
2. THE Game_Page SHALL display the trivia question text prominently below the song information
3. THE Game_Page SHALL display the 4 answer options as tappable buttons arranged vertically
4. THE Game_Page SHALL display the leaderboard in a collapsible section so it does not obscure the question area
5. THE Game_Page SHALL be responsive and usable on mobile devices with a minimum viewport width of 320px
6. THE Game_Page SHALL display the album art of the currently playing song
