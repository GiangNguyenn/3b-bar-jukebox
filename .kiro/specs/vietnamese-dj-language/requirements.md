# Requirements Document

## Introduction

This feature adds Vietnamese as a selectable language option for DJ Mode. When Vietnamese is selected, the DJ script is generated in Vietnamese using an AI model well-suited for Vietnamese language output, and the text-to-speech audio is synthesised using a TTS model that supports Vietnamese. The language selection is persisted per-admin session and integrates cleanly into the existing DJ Mode settings area without disrupting the default English behaviour.

## Glossary

- **DJ_Mode**: The existing optional feature that triggers AI-generated DJ announcements between tracks (see dj-mode spec).
- **DJ_Language**: The admin-configured language in which the DJ_Script is generated and spoken. Valid values: `"english"` (default) and `"vietnamese"`.
- **DJ_Script**: A short spoken announcement generated by an AI model that introduces the next track and artist.
- **DJService**: The existing singleton service (`services/djService.ts`) that manages DJ announcement logic, prefetching, and audio playback.
- **Script_API**: The existing server-side route `/api/dj-script` that proxies AI script generation requests.
- **TTS_API**: The existing server-side route `/api/dj-tts` that proxies TTS audio synthesis requests.
- **Venice_AI**: The third-party LLM API (venice.ai) used for English DJ script generation.
- **Vietnamese_LLM**: The AI model used to generate DJ scripts in Vietnamese. Hardcoded constant `qwen3-235b-a22b-instruct-2507` in the Script_API route.
- **Vietnamese_TTS**: The TTS model used to synthesise Vietnamese speech. Hardcoded constant `tts-qwen3-1-7b` in the TTS_API route. Unlike `tts-kokoro`, Qwen3 TTS uses language-aware synthesis and does not accept named voice parameters.
- **VENICE_AI_API_KEY**: The existing environment variable holding the API key for Venice AI services.
- **Admin_Page**: The admin control panel at `app/[username]/admin/` where the operator manages the jukebox.
- **localStorage**: The browser storage mechanism used to persist admin settings across page refreshes.

## Requirements

### Requirement 1: Language Selection UI

**User Story:** As an admin, I want to select the DJ language from the admin dashboard, so that I can switch between English and Vietnamese DJ announcements.

#### Acceptance Criteria

1. THE Admin_Page SHALL display a language selector labeled "DJ Language" in the dashboard tab settings area, visible only when DJ Mode is enabled.
2. THE DJ Language selector SHALL offer exactly two options: "English" (default) and "Vietnamese".
3. WHEN the admin selects a DJ Language option, THE Admin_Page SHALL persist the selection to `localStorage` under the key `"djLanguage"` so it survives page refreshes.
4. THE Admin_Page SHALL read the persisted `"djLanguage"` value from `localStorage` on load and display the matching option as selected.
5. IF no `"djLanguage"` value is present in `localStorage`, THEN THE Admin_Page SHALL default to "English".
6. WHEN DJ Mode is disabled, THE Admin_Page SHALL hide the DJ Language selector to avoid confusion.

---

### Requirement 2: Vietnamese Script Generation

**User Story:** As an admin, I want the DJ script to be generated in Vietnamese when Vietnamese is selected, so that Vietnamese-speaking guests hear a natural-sounding announcement.

#### Acceptance Criteria

1. WHEN a DJ announcement is triggered and DJ_Language is `"vietnamese"`, THE Script_API SHALL generate the DJ_Script in Vietnamese using the hardcoded Vietnamese_LLM model identifier `qwen3-235b-a22b-instruct-2507`.
2. WHEN a DJ announcement is triggered and DJ_Language is `"english"`, THE Script_API SHALL generate the DJ_Script in English using the existing Venice_AI model, preserving all current behaviour.
3. THE Script_API SHALL accept a `language` field in the request body alongside the existing `trackName`, `artistName`, and `recentScripts` fields.
4. WHEN DJ_Language is `"vietnamese"`, THE Script_API SHALL use a system prompt instructing the model to write a short, natural Vietnamese DJ announcement of no more than 2 sentences.
5. THE DJService SHALL read the current `"djLanguage"` value from `localStorage` and include it in every request sent to the Script_API.
6. IF the `language` field is absent from the Script_API request body, THEN THE Script_API SHALL default to English behaviour, preserving backward compatibility.

---

### Requirement 3: Vietnamese Text-to-Speech

**User Story:** As an admin, I want the DJ audio to be spoken in Vietnamese when Vietnamese is selected, so that the voice matches the language of the script.

#### Acceptance Criteria

1. WHEN a DJ_Script is generated in Vietnamese, THE DJService SHALL send the script to the TTS_API with a `language` field set to `"vietnamese"`.
2. WHEN the TTS_API receives a request with `language` set to `"vietnamese"`, THE TTS_API SHALL call the Venice AI `/audio/speech` endpoint using the hardcoded model identifier `tts-qwen3-1-7b` with `model`, `input`, and `response_format` fields only — no `voice` parameter, as Qwen3 TTS uses language-aware synthesis without named voices.
3. WHEN the TTS_API receives a request with `language` set to `"english"` or with no `language` field, THE TTS_API SHALL use the existing `tts-kokoro` model and `af_nova` voice, preserving all current behaviour.
4. THE TTS_API SHALL return the synthesised audio as an `audio/mpeg` binary stream regardless of the selected language.
5. IF the Vietnamese_TTS model is unavailable, THEN THE TTS_API SHALL return a 500 error and THE DJService SHALL skip the announcement and play the Next_Track immediately.

---

### Requirement 4: Graceful Degradation

**User Story:** As an admin, I want the DJ feature to continue working even if Vietnamese generation fails, so that music playback is never interrupted.

#### Acceptance Criteria

1. IF the Vietnamese_LLM request fails or returns an error, THEN THE DJService SHALL log the error and proceed to play the Next_Track without an announcement.
2. IF the Vietnamese_TTS request fails or returns an error, THEN THE DJService SHALL log the error and proceed to play the Next_Track without an announcement.
3. WHEN any error occurs in the Vietnamese language path, THE DJService SHALL resolve `maybeAnnounce` without throwing, preserving the existing graceful fallback invariant.
4. IF DJ_Language is set to an unrecognised value in `localStorage`, THEN THE DJService SHALL treat it as `"english"` and use the default English path.

---

### Requirement 5: Backward Compatibility

**User Story:** As an admin, I want the existing English DJ Mode to continue working exactly as before after this feature is added, so that no existing behaviour is broken.

#### Acceptance Criteria

1. WHEN DJ_Language is `"english"` or absent, THE Script_API SHALL behave identically to its pre-feature behaviour, using the same model, prompt, and request structure.
2. WHEN DJ_Language is `"english"` or absent, THE TTS_API SHALL behave identically to its pre-feature behaviour, using `tts-kokoro` and `af_nova`.
3. THE DJService SHALL pass the `language` field to both the Script_API and TTS_API on every request, defaulting to `"english"` when `"djLanguage"` is absent from `localStorage`.
4. THE existing `djMode`, `djFrequency`, and `duckOverlayMode` localStorage keys SHALL remain unchanged in name, type, and semantics.
